Архитектурный анализ процесса обучения кастомных авторегрессионных трансформеров с использованием FlashAttention, Rotary Position Embeddings (RoPE) и методов глубокой оптимизации памяти.

---

## 📊 Эволюция архитектуры: V9 vs V10

В данном проекте задокументирован классический инженерный компромисс при создании LLM: масштабирование емкости модели против сохранения качества сходимости при жестких аппаратных ограничениях.

| Характеристика / Метрика | Версия 9 (Стабильный бейзлайн) | Версия 10 (Экспериментальная) |
| :--- | :--- | :--- |
| **Параметры** | ~130M ($d_{model}=768$, 32 слоя) | ~300M ($d_{model}=1024$, 32 слоя) |
| **Attention** | Native PyTorch SDPA | Оптимизированная SDPA с явным RoPE |
| **Оптимизатор** | FP32 `AdamW` (Высокая точность) | `AdamW8bit` (Оптимизация памяти) |
| **LR Scheduling** | Constant ($1e-4$) | Cosine Decay с Warmup ($3e-4 \to 3e-5$) |
| **Batch Config** | Эфф. Batch: 32 ($4 \times 8$) | Эфф. Batch: 64 ($1 \times 64$) |
| **Качество генерации** | 🏆 **Высокая связность и стабильность** | Сниженное качество (возможен underfitting) |

## 🔬 Инженерный разбор (Post-Mortem)

### 1. Проблема утечки VRAM в версии V9
* **Проблема:** Возобновление обучения из чекпоинта приводило к падению производительности в 10 раз и раздуванию памяти.
* **Причина:** Стандартный FP32 `AdamW` выделяет тензоры моментов (momentum), равные $2 \times$ количеству параметров модели. Перезагрузка `state_dict` вызывала критическую фрагментацию CUDA-памяти на GPU.

### 2. Почему V10 показала себя хуже несмотря на архитектуру
* **Несоответствие данных:** Увеличение размера эмбеддингов ($768 \to 1024$) резко повысило емкость модели. Для малых датасетов (`input.txt`) потребовалось бы значительно больше токенов для качественного обучения латентного пространства.
* **Шум квантования:** Использование `AdamW8bit` решило проблему памяти, но шум квантования в градиентах привел к деградации точности предсказания токенов.
* **Дисперсия градиента:** Снижение микро-батча до $1$ (с 64 шагами накопления) увеличило дисперсию градиента, что в сочетании с 8-битной оптимизацией привело к нестабильным обновлениям весов.

## 🛠 Реализованные возможности
* **RoPE (Rotary Position Embeddings):** Замена абсолютных позиционных кодировок на относительные ротационные для лучшей экстраполяции контекста.
* **Управление памятью:** Интеграция PyTorch Gradient Checkpointing (`use_reentrant=False`) для всех 32 слоев.
* **Стабильность:** Архитектура Pre-LayerNorm с использованием `RMSNorm` вместо стандартного `LayerNorm` для защиты от взрыва градиентов.
